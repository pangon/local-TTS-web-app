# GOAL-backend-monitoring: Backend Processing Monitoring

**Description**: Expose detailed information about local backend processing through the web interface. Users and operators can monitor what the system is doing, track job progress, and diagnose issues.

**Status**: Approved

**Priority**: Should-have

**Source stakeholder**: [STK-self-hoster](../stakeholders.md)

## Success Criteria

- [ ] Users can view the status of ongoing TTS/audiobook processing jobs
- [ ] Resource usage and model information are visible
- [ ] Processing errors and logs are accessible from the UI
